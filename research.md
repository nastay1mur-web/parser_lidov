# Исследование: Telegram Mini App — Парсер лидов
> Дата: март 2026 | Источники: leadscanbot.ru, leadscannerbot.com, fl-bot.ru, fbot.pro, vakas-tools.ru, core.telegram.org

---

## Концепция нашего продукта

Пользователь (фрилансер / агентство) открывает Telegram-бота и видит список потенциальных клиентов, которые публично написали в соцсетях что ищут разработчика, дизайнера, маркетолога и т.д.
По каждому лиду есть **прямая ссылка** → нажал → попал в чат/пост → написал клиенту.

## 5 аналогов: анализ конкурентов

---

### 1. LeadScanner AI
**Сайт:** leadscannerbot.com | **Формат:** Telegram-бот + веб-кабинет

**Что делает:**
AI анализирует сообщения в Telegram-чатах в реальном времени. Пользователь описывает бизнес на естественном языке — AI сам определяет ключевые слова, регионы и целевую аудиторию. Лиды приходят в Telegram мгновенно.

**Сильные стороны:**
- Настройка за 2 минуты — не нужно вручную вводить ключевые слова
- AI фильтрует спам, любопытных и ботов — только горячие заявки
- Слоты по нишам: можно отслеживать несколько направлений одновременно
- Уведомление содержит: категорию, ключевое слово, текст, ник, время — всё в одном сообщении
- Тарифы понятны: Base $15 / Pro $35 / Business $75

**Слабые стороны:**
- Нет Mini App — только бот и сайт, неудобно управлять с телефона
- Нет истории лидов внутри Telegram — надо заходить на сайт
- Нет шаблонов ответов — пользователь сам придумывает текст
- Нет фильтрации по бюджету/гео внутри бота
- Нет статистики (сколько лидов получено, сколько отвечено)

**Что мы можем сделать лучше:**
- Mini App с лентой лидов прямо в Telegram — не нужен сайт
- Шаблоны ответов с авто-подстановкой имени и ключевого слова
- История + статус лида (новый / написал / ответил)

### 2. LeadScanBot
**Сайт:** leadscanbot.ru | **Формат:** Telegram-бот

**Что делает:**
Мониторит открытые и закрытые Telegram-чаты по ключевым словам. Пользователь добавляет подписку, настраивает ключевые слова и слова-исключения, выбирает чаты. Уведомления приходят в течение нескольких секунд.

**Сильные стороны:**
- Работает с закрытыми чатами — расширяет охват
- Блокировка нежелательных пользователей
- Централизованная система заявок — всё в одном боте
- Широкий охват ниш: недвижимость, HR, B2B, фриланс, e-commerce

**Слабые стороны:**
- Интерфейс — только текстовые команды бота, неудобно на мобильном
- Нет визуального списка лидов — приходят как обычные сообщения в чат
- Тяжело управлять несколькими подписками в текстовом интерфейсе
- Нет прямой кнопки "Ответить" — нужно самому копировать ник и открывать чат
- Нет шаблонов и статистики

**Что мы можем сделать лучше:**
- Карточки лидов с кнопкой "Написать" в один клик
- Визуальный интерфейс управления подписками (Mini App)
- Фильтр лидов по источнику, нише, дате

### 3. Find Lead Bot (@find_lead_bot by vakas-tools.ru)
**Формат:** Telegram-бот | **Позиционирование:** биржа лидов в реальном времени

**Что делает:**
Находит заявки в Telegram — не реклама и не холодный поиск, а реальные люди которые сами написали что им нужна услуга. Заявки появляются сами по мере поступления в чаты.

**Сильные стороны:**
- Простота входа — /start и сразу можно получать лиды
- Бесплатный тариф для старта
- Лиды "горячие" — человек сам написал запрос
- Подходит для фрилансеров без рекламного бюджета

**Слабые стороны:**
- Нет Mini App — только бот-интерфейс
- Лиды приходят потоком без возможности фильтрации на лету
- Нет сохранения истории и CRM-функций
- Нет группировки по нишам/источникам
- Нельзя пометить лид как обработанный

**Что мы можем сделать лучше:**
- Лента с фильтрами по нише, источнику, дате
- Статусы лидов: новый / написал / ответил / закрыт
- Закреплять "горячих" лидов вверху списка

### 4. Freelance Bot (fl-bot.ru)
**Сайт:** fl-bot.ru | **Формат:** Telegram-бот + шаблоны откликов

**Что делает:**
Агрегирует новые заказы с FL.ru, Kwork, Freelance.ru. Приходят уведомления быстрее официальных (10–30 сек vs 2–5 мин). К каждому заказу прилагается **готовый шаблон отклика** для данной категории.

**Сильные стороны:**
- Скорость — опережает конкурентов на 2-5 минут, это критично
- Готовые тексты откликов — огромная ценность для новичков
- Три фриланс-биржи в одном месте
- Понятные тарифы (199₽/месяц)
- Фильтрация по специальности и бюджету

**Слабые стороны:**
- Только фриланс-биржи — не парсит соцсети (VK, Telegram-чаты)
- Шаблоны общие, не персонализированные
- Нет Mini App — управление через сайт и бот отдельно
- Нет аналитики (какой процент откликов получил ответ)
- Нет истории заказов внутри бота

**Что мы можем сделать лучше:**
- Источники шире: Telegram-чаты + VK + форумы
- Шаблоны с подстановкой {имя}, {ключевое_слово} — более персональные
- История обращений с отметкой результата

### 5. FBot (fbot.pro)
**Сайт:** fbot.pro | **Формат:** Telegram-бот + веб-кабинет + автоответы

**Что делает:**
Мониторит FL, Freelance, Freelancehunt, Weblancer, Kwork. Уникальная функция — **автоответы**: бот сам отправляет заготовленный текст заказчику в течение 3 минут после публикации.

**Сильные стороны:**
- Автоответы — революционная функция, первый отклик без участия пользователя
- Самый широкий охват бирж (5 платформ)
- 1000 уведомлений бесплатно — низкий барьер входа
- Настройка фильтров: категории, ключевые слова, бюджет

**Слабые стороны:**
- Автоответы только для FL.ru, дорого (1270₽/мес)
- Настройки только через сайт — неудобно с телефона
- Нет Mini App
- Нет аналитики по откликам
- Автоответы могут выглядеть спамом без персонализации

**Что мы можем сделать лучше:**
- Шаблоны с AI-персонализацией под каждый запрос
- Одно место для всего: настройки + лента + история — прямо в Telegram

### 6. @kwork_bot (официальный бот Kwork.ru)
**Формат:** Telegram-бот (без Mini App)

**Что делает:**
Уведомляет о новых заказах на Kwork.ru по выбранным категориям. Кнопка "Откликнуться" ведёт на страницу заказа на сайте.

**Сильные стороны:**
- Мгновенные push-уведомления без захода на сайт
- Авторизация через существующий профиль Kwork
- Знакомая аудитория — платформа уже известна фрилансерам
- Простота: одна кнопка действия

**Слабые стороны:**
- Нет Mini App UI — только текст и inline-кнопки, примитивный интерфейс
- Нет фильтрации по бюджету / ключевым словам прямо в боте
- Только заказы внутри Kwork — никаких внешних источников
- Нет CRM: пометить, отложить, добавить заметку — невозможно
- Отклик требует выхода на сайт — прямого контакта нет

**Что мы делаем лучше:**
- Лиды из соцсетей, где клиент ещё не разместил заказ на бирже → 0 конкурентов
- Прямая ссылка на человека без посредника
- Mini App с карточками, статусами и шаблонами

### 7. @flru_bot (официальный бот FL.ru)
**Формат:** Telegram-бот (без Mini App)

**Что делает:**
Присылает уведомления о новых проектах FL.ru. Карточка содержит название, категорию, бюджет, дедлайн, число откликов. Для отклика — переход на сайт.

**Сильные стороны:**
- Детальные карточки: бюджет, дедлайн, уже поданные отклики
- Фильтрация по специализации прямо в боте
- Большое покрытие: одна из крупнейших русскоязычных бирж
- Уведомления приходят быстро

**Слабые стороны:**
- Перенаправляет на сайт для любого действия — бот только "витрина"
- Нет агрегации из Telegram-каналов и соцсетей
- Нет информации о клиенте внутри карточки (кто он, его история)
- UX устарел, не обновлялся годами
- Комиссия платформы 15–20% с каждой сделки

**Что мы делаем лучше:**
- Клиент ещё не на бирже → комиссия 0%, конкуренция 0%
- Информация об источнике (откуда пришёл лид) + прямая ссылка на профиль

## Экраны Mini App (обязательные)

### 1. Онбординг / Настройка (первый запуск)
Показывается один раз. Цель: получить от пользователя нишу и ключевые слова.
- Поле "Опишите чем занимаетесь" (свободный текст)
- Мультивыбор специальностей (разработчик / дизайнер / маркетолог / копирайтер...)
- Предложенные ключевые слова на основе выбора
- BottomButton "Начать поиск"

### 2. Лента лидов (главный экран)
Основной экран, открывается при каждом входе.
- Таббар вверху или фильтр-чипсы: Все / Новые / Telegram / VK / Форумы
- Карточки лидов (см. ниже)
- Pull-to-refresh
- Бейдж с количеством новых лидов
- Пустое состояние ("Лидов пока нет, ищем...")

### 3. Карточка лида (inline в ленте)
Каждый лид — это карточка с:
- Аватар + имя/ник пользователя
- Иконка источника (Telegram / VK / Reddit) + название чата/группы
- Ключевое слово (тег)
- Текст сообщения (2-3 строки, кнопка "Читать полностью")
- Время публикации (5 мин назад / 2 часа назад)
- Кнопка **"Написать"** → открывает SendMessage или прямую ссылку
- Кнопка **"Горячий"** (огонёк) — закрепить
- Кнопка **"Нецелевой"** (крест) — скрыть

### 4. Детали лида (bottom sheet или отдельный экран)
Открывается по тапу на карточку.
- Полный текст сообщения
- Все доступные контакты (TG / VK / телефон)
- История взаимодействий с этим лидом
- Статус: новый / написал / ответил / закрыт
- Кнопки: Написать / Сохранить / Нецелевой
- Заметка (текстовое поле)

### 5. Написать сообщение (SendMessage)
Открывается как bottom sheet поверх деталей.
- Табы: Email | Telegram | WhatsApp | VK
- Выбор шаблона (dropdown)
- Текст с подстановкой {имя} и {ключевое_слово}
- BottomButton "Открыть чат" → deep link

### 6. Настройки
- Ключевые слова (добавить / удалить теги)
- Источники (включить / выключить: TG, VK, Reddit, форумы)
- Стоп-слова
- Уведомления (пуш / тихий режим)
- Фильтр по бюджету/гео

### 7. Шаблоны
- Список шаблонов по категориям: Первый контакт / Предложение / Фолоу-ап
- Редактор с переменными {имя}, {ключевое_слово}, {источник}
- Создать / Редактировать / Удалить шаблон

### 8. Аналитика
- Лидов за период (день / неделя / месяц)
- Распределение по источникам (pie chart)
- Конверсия: написал → ответил → сделка
- Топ ключевых слов

## Обязательные UI-элементы

### Telegram-native элементы (через WebApp API)
| Элемент | Использование |
|---------|--------------|
| `BackButton` | Навигация назад на всех вложенных экранах |
| `BottomButton` | Главный CTA: "Начать поиск", "Открыть чат", "Сохранить" |
| `SecondaryButton` | Второй action: "Нецелевой" рядом с "Написать" |
| `HapticFeedback` | При лайке, смахивании, отправке сообщения |
| `setHeaderColor()` | Адаптация к теме пользователя (light/dark) |
| `themeParams` CSS-переменные | Цвета кнопок, текста, фонов через `var(--tg-theme-*)` |

### UI-компоненты в карточке лида
- Аватар (заглушка если нет фото)
- Иконка источника (цветная: синяя для TG, голубая для VK)
- Бейдж статуса (Новый / Горячий)
- Timestamp в относительном формате ("5 мин назад")
- Две кнопки действия: иконочные, не текстовые (экономия места)
- Свайп-жест вправо = "Написать", влево = "Скрыть"

### Обязательные состояния каждого экрана
- Loading skeleton (не спиннер — скелетон карточек)
- Empty state с объяснением и CTA
- Error state с кнопкой "Попробовать снова"
- Success toast после действия

**Выбор: Bottom Tab Bar** (не свайп-карточки, не only-список)

## Цвета и тема

```css
/* Telegram CSS-переменные — автоматически адаптируются к теме пользователя */
--tg-theme-bg-color           /* основной фон */
--tg-theme-secondary-bg-color /* фон карточек, секций */
--tg-theme-text-color         /* основной текст */
--tg-theme-hint-color         /* метаданные: время, источник */
--tg-theme-link-color         /* ссылки */
--tg-theme-button-color       /* цвет кнопок */
--tg-theme-button-text-color  /* текст на кнопках */

/* Цвета платформ — фиксированные брендовые */
--color-telegram: #2AABEE
--color-vk:       #4C75A3
--color-reddit:   #FF4500

/* Статусы лидов */
--lead-new:  #3b82f6   /* синий — новый */
--lead-hot:  #f97316   /* оранжевый — горячий */
--lead-done: #22c55e   /* зелёный — обработан */
--lead-skip: #6b7280   /* серый — нецелевой */
```

**Светлая тема:** фон `#EFEFF4`, карточки `#FFFFFF`, текст `#000000`, hint `#8E8E93`
**Тёмная тема:** фон `#1C1C1E`, карточки `#2C2C2E`, текст `#FFFFFF`, hint `#8E8E93`

## Микровзаимодействия и Haptic Feedback

```javascript
const haptic = Telegram.WebApp.HapticFeedback;

haptic.impactOccurred('light');    // тап на карточку → переход
haptic.impactOccurred('medium');   // сохранение лида ♡
haptic.notificationOccurred('success');  // лид обработан
haptic.notificationOccurred('warning');  // лид скрыт
haptic.notificationOccurred('error');    // ошибка / нет сети
haptic.selectionChanged();               // свайп-действие началось
```

### Swipe-to-action на карточках (паттерн iOS Mail)
```
Свайп влево  → [Скрыть] [В работе]
Свайп вправо → быстрое сохранение (♡)
```

### Анимация появления карточек
```css
@keyframes slideUp {
  from { transform: translateY(20px); opacity: 0; }
  to   { transform: translateY(0);    opacity: 1; }
}
.lead-card:nth-child(1) { animation: slideUp 0.2s ease-out 0ms both; }
.lead-card:nth-child(2) { animation: slideUp 0.2s ease-out 50ms both; }
.lead-card:nth-child(3) { animation: slideUp 0.2s ease-out 100ms both; }
```
Правило: длительность 150–250ms. Быстрее — незаметно, медленнее — раздражает.

## Критические правила Telegram Mini App

### НЕЛЬЗЯ
- `alert()` / `confirm()` / `prompt()` → только `WebApp.showAlert()`, `showConfirm()`, `showPopup()`
- `window.open()` → только `WebApp.openLink()` или `openTelegramLink()`
- Шрифты меньше 12px
- Кнопки меньше 44px по высоте (минимальный touch target)
- Игнорировать safe area:
```css
padding-bottom: env(safe-area-inset-bottom);
```

### ОБЯЗАТЕЛЬНО при запуске
```javascript
Telegram.WebApp.expand();  // развернуть на весь экран
Telegram.WebApp.ready();   // скрыть нативный loader
```

- [ ] Иконки платформ (TG/VK/Reddit) цветные и различимые
- [ ] Badge "NEW" на лидах свежее 2-3 часов

## Экспертная оценка

---

### Взгляд UX-дизайнера

#### Типичные ошибки новичков в подобных приложениях

**1. "Информационная свалка" в карточке**
Новички пытаются уместить всё — полный текст, все контакты, все метаданные — в одну карточку в ленте. Результат: карточка занимает весь экран, пользователь видит 1-2 лида вместо 5-6 и быстро устаёт.
Правило: в ленте — минимум (кто, откуда, суть в 2 строках, 2 кнопки). Всё остальное — на детальном экране.

**2. Отсутствие пустых состояний**
"Лиды не найдены" — и белый экран. Пользователь не понимает: сломалось? не настроено? нет подписки?
Каждый пустой экран должен объяснять причину и давать действие: "Добавьте ключевые слова в настройках → [Настроить]".

**3. Неразличимые статусы лидов**
Новый лид выглядит так же как просмотренный. Пользователь листает всё подряд, теряет фокус, не понимает что уже обработано.
Обязательно: визуальное различие новый/просмотренный (жирный текст vs обычный, цветная точка vs серая).

**4. Кнопка "Написать" ведёт не туда**
Пользователь нажимает — открывается браузер, а не Telegram. Или открывается профиль, а не чат.
Нужно `openTelegramLink()` для t.me-ссылок и `openLink()` для остальных. Разница критическая.

**5. Нет онбординга — и нет результата**
Пользователь устанавливает, видит пустую ленту и уходит. Он не понял что нужно настроить ключевые слова.
Онбординг — не туториал из 5 слайдов. Это один экран: "Введите свою специальность → покажем первые лиды".

**6. Перегруженные настройки**
Большой экран настроек с 20 полями пугает. Пользователь не трогает — и не получает лиды.
Принцип прогрессивного раскрытия: базовые настройки сразу, расширенные — за раскрывашкой "Дополнительно".

---

#### Что обязательно должно быть

| Элемент | Почему |
|---------|--------|
| Счётчик новых лидов при открытии | Мотивирует открыть приложение |
| Визуальный индикатор "новый" на карточке | Фокус внимания на необработанных |
| Кнопка "Написать" прямо в ленте (не только в деталях) | Скорость — главная ценность продукта |
| Относительное время ("5 мин назад") | Горячесть лида определяется временем |
| Фильтр "только новые" в один тап | 80% сессий — просмотр новых лидов |
| Статус лида (новый / написал / ответил) | Без CRM-минимума продукт теряет ценность |

---

#### Что бесит пользователей (на основе анализа конкурентов)

- Дублирующиеся лиды — один и тот же человек из 3 разных чатов → нужна дедупликация по нику/телефону
- Спамные лиды — "ищу что-нибудь" без конкретики → нужны стоп-слова и AI-фильтр
- Нет возможности вернуться к лиду, который уже промотал → нужна вкладка "Просмотренные" или "Сохранённые"
- Уведомление пришло, открыл — лид уже исчез (удалён в источнике) → показывать кешированный текст
- Каждый раз нужно заходить на сайт для настройки → всё должно быть в Mini App
- Нет подтверждения что сообщение отправлено → toast "Чат открыт" после нажатия "Написать"

### Взгляд маркетолога

#### Что заставит попробовать

**Главный крючок: результат до оплаты**
Пользователь должен получить первого реального лида до того как увидит экран оплаты. Сценарий: онбординг → 3 бесплатных лида сразу → "Хотите больше? 199₽/месяц". Человек уже видел реальных клиентов — конверсия в платящего кратно выше.

**Позиционирование через скорость, не через функции**
Не "парсер лидов с фильтрами и шаблонами" — а "Узнай о клиенте раньше конкурентов". Скорость (лид за 5 минут до остальных) — это деньги. Именно это надо продавать.

**Социальное доказательство прямо в интерфейсе**
Показывать в ленте: "За последние 24 часа пользователи нашли 847 лидов". Или счётчик в онбординге: "Уже 2 300 фрилансеров получают клиентов". Живые цифры снимают недоверие.

---

#### Элементы повышающие конверсию в подписку

**1. Freemium с жёстким но честным ограничением**
Бесплатно: 5 лидов в день, только 2 источника, нет шаблонов.
Платно: безлимит, все источники, шаблоны, история, аналитика.
Ограничение должно быть ощутимым, но не блокирующим — пользователь видит ценность и хочет больше.

**2. Счётчик "сколько лидов вы пропустили"**
На бесплатном тарифе: "Сегодня появилось ещё 12 лидов — вы видите только 5. [Открыть все →]". Работает как FOMO (страх упустить).

**3. Уведомление-триггер**
Пуш в нужный момент: "Новый лид — ищет разработчика, написал 3 минуты назад". Пользователь открывает — видит реального человека. Это лучший момент для конверсии: "Хотите получать такие уведомления без задержки? Подключите Pro".

**4. Быстрая победа в онбординге**
Онбординг должен заканчиваться не экраном "Настройка завершена", а показом первых лидов. Человек сразу видит что продукт работает — это снимает главное возражение "а вдруг там пусто".

**5. Реферальная механика**
"Пригласи коллегу → +7 дней Pro бесплатно". Реализуется через Telegram-инвайт ссылку, не требует email. Виральность в Telegram работает хорошо.

**6. Напоминание о просмотренных лидах**
Пуш через 4 часа: "Вы не ответили на 3 лида из сегодняшней ленты. Клиенты ждут." Возвращает пассивных пользователей.

---

#### Удержание (Retention)

- **Streak**: "7 дней подряд находишь клиентов" — геймификация без лишнего шума
- **Еженедельный дайджест**: "На этой неделе: 34 лида, 8 контактов, 2 в переговорах" — в личный чат бота
- **Пустая лента = проблема настроек**, не проблема продукта — объяснять это явно

### Взгляд Telegram-разработчика

#### Технические ограничения которые важно знать заранее

**1. Mini App — это веб-приложение внутри Telegram**
Стек: HTML + CSS + JS (React/Vue/Svelte — любой). Никакого доступа к нативным API устройства кроме того что даёт WebApp API. Камера, файловая система, Bluetooth — недоступны.

**2. Размер и скорость загрузки**
Mini App загружается каждый раз заново (нет постоянного процесса). Пользователь открывает — ждёт загрузку. Оптимальный размер bundle: до 200KB gzip. Больше — заметная задержка на медленном интернете.
Решение: code splitting, lazy loading деталей лида, кеширование данных в localStorage.

**3. MainButton — один на экран**
Только один MainButton и один SecondaryButton одновременно. Нельзя сделать "тулбар с 4 кнопками" внизу через нативный API — только через собственный CSS.
Важно: MainButton перекрывает нижнюю часть контента. Нужен `padding-bottom` равный высоте кнопки (~50px).

**4. Открытие ссылок**
```javascript
// Ссылки t.me — открываются в Telegram (не браузере)
Telegram.WebApp.openTelegramLink('https://t.me/username');

// Внешние ссылки — в браузере или in-app
Telegram.WebApp.openLink('https://vk.com/...');

// window.open() — НЕ работает на iOS Telegram
```

**5. Авторизация пользователя**
Telegram передаёт `initData` при открытии Mini App. Это подписанные данные с ID, именем, username пользователя. Проверяй подпись на бэкенде через HMAC-SHA256 — иначе кто угодно может подделать запрос.
```javascript
const user = Telegram.WebApp.initDataUnsafe.user;
// user.id, user.first_name, user.username
```

**6. Клавиатура меняет viewport**
При открытии поля ввода (поиск, заметка) клавиатура поднимается и меняет высоту `window`. Обязательно слушать:
```javascript
Telegram.WebApp.onEvent('viewportChanged', () => {
  // пересчитать высоты, прокрутить к активному полю
});
```

**7. Закрытие Mini App**
Пользователь может закрыть приложение в любой момент. Нет события "перед закрытием" (как `beforeunload`). Все несохранённые данные теряются. Решение: автосохранение в localStorage после каждого действия.

---

#### Что работает хорошо

| Возможность | Как использовать |
|------------|-----------------|
| `HapticFeedback` | Делает интерфейс "живым", пользователи замечают |
| `showPopup` / `showConfirm` | Нативные диалоги — выглядят как часть Telegram |
| `themeParams` | Автоматическая тёмная/светлая тема без усилий |
| `CloudStorage` | 1024 байт постоянного хранилища на стороне Telegram (не localStorage) |
| `openTelegramLink` | Плавный переход в чат без выхода из экосистемы |
| Уведомления через бота | Бот может слать пуши пока Mini App закрыт — это основной канал удержания |

#### Что лучше не делать

| Антипаттерн | Почему плохо |
|------------|-------------|
| Сложный onboarding из 5+ экранов | Пользователи закрывают Mini App — возврат низкий |
| Загрузка всех лидов сразу | Долго + лагает. Нужна пагинация (20 лидов за раз) |
| Хранить всё в localStorage | Очищается браузером. Критические данные — на бэкенд |
| Кастомный скролл через JS | Дёргается, лагает. Использовать нативный CSS overflow |
| Анимации > 300ms | Ощущение тормозов. До 250ms — норма |
| Шрифты через @import Google Fonts | Доп. запрос = задержка. Использовать системные шрифты |
| Много запросов при старте | Один запрос на загрузку ленты — всё остальное лениво |
