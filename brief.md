# Brief: Telegram Mini App — Парсер лидов
> Версия 1.0 | Март 2026 | Основан на research.md

---

## Концепция одной фразой
Открыл бота → увидел людей которые прямо сейчас ищут твои услуги → нажал "Написать" → попал в чат с клиентом.

---

## Дизайн-система

### Стиль: нативный Telegram (MAX)
Приложение должно ощущаться как встроенная часть Telegram — не сайт, не веб-страница.

```
Цвета:     var(--tg-theme-*)  — берём у Telegram автоматически
Шрифты:    системные (SF Pro на iOS, Roboto на Android)
Углы:      12–16px (как в нативных компонентах Telegram)
Тени:      минимальные, только для карточек
Анимации:  150–250ms ease-out
```

### Обязательные нативные элементы
| Элемент | Использование |
|---------|--------------|
| `Telegram.WebApp.MainButton` | Главный CTA каждого экрана (синяя кнопка снизу) |
| `Telegram.WebApp.BackButton` | Стрелка назад в хедере Telegram |
| `Telegram.WebApp.HapticFeedback` | Вибрация на ключевых действиях |
| `Telegram.WebApp.showPopup` | Подтверждение удаления, ошибки |
| `Telegram.WebApp.openTelegramLink` | Открытие чата с клиентом |

### Типографика (вся иерархия)
```
Заголовок экрана:  17px / 600 / text-color
Имя/ник лида:      15px / 600 / text-color
Текст сообщения:   14px / 400 / text-color
Метаданные:        12px / 400 / hint-color
Теги/чипы:         11px / 500 / hint-color
```

#### Элементы
- Иконка 📡 (64px, по центру)
- Заголовок: "Находи клиентов раньше конкурентов"

**Цель:** быстро просмотреть новых лидов и написать самым интересным

│ │🔷 @anna_s · ВКонтакте   │ │

│ │ Нужен SMM-специалист    │ │

│ │ [Написать →]      [♡]   │ │

│ ┌─────────────────────────┐ │
│ │ ░░░░░░░░░░░░░░░░░░░░░░ │ │  ← скелетон при загрузке
│ │ ░░░░░░░░░░░░░░░░░░░░░░ │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

#### Элементы

**Хедер (нативный)**
- Заголовок: "Лиды"
- Иконка 🔍 (поиск) — справа
- Иконка ⚙️ (настройки) — справа

**Фильтр-чипы** (горизонтальный скролл, высота 32px)
- "Все" (активный = синий фон)
- "TG" с иконкой Telegram
- "VK" с иконкой VK
- "Форумы"

**Плашка новых лидов** (показывается если есть непросмотренные)
- Фон: синий/оранжевый
- Текст: "🔴 N новых лида"
- Нажатие: скролл к первому новому

**Карточка лида** (высота ~100px):
- Строка 1: Иконка источника (цветная, 16px) + ник (жирный) + источник + время
- Строка 2: Тег специальности (чип) + индикатор "🔥" если < 30 мин
- Строка 3–4: Текст (2 строки, обрезается)
- Строка 5: Кнопка "Написать →" (акцентный синий) + иконка ♡

**Индикатор непрочитанного**: синяя точка 8px слева от карточки (только у новых)

#### Действия
| Действие | Результат |
|----------|-----------|
| Тап на карточку | → Экран 2 (Детали лида) |
| Тап "Написать →" | → Экран 3 (Отправить сообщение) |
| Тап ♡ | Сохранить лид, иконка → оранжевая, haptic medium |
| Тап фильтр-чипа | Фильтрация ленты, haptic selection |
| Тап 🔍 | → Экран 4 (Поиск) |
| Тап ⚙️ | → Экран 5 (Настройки) |
| Pull-to-refresh | Обновление ленты, haptic light |
| Свайп влево на карточке | Появляются кнопки [В работе] [Скрыть] |
| Свайп вправо на карточке | Быстрое сохранение ♥, haptic medium |
| Тап "Загрузить ещё" | Подгрузка следующих 20 лидов |

#### Переходы
```
Тап карточки          → Экран 2 (push справа налево)
Тап "Написать →"      → Экран 3 (bottom sheet снизу)
Тап 🔍               → Экран 4 (push)
Тап ⚙️               → Экран 5 (push)
Bottom tab "Горячие"  → Экран 1б (Горячие лиды)
Bottom tab "Настройки"→ Экран 5
```

#### Состояния
- **Loading**: skeleton-карточки (3 штуки, анимация pulse)
- **Empty**: иллюстрация + "Лидов пока нет" + кнопка "Настроить поиск"
- **Error**: "Нет соединения" + кнопка "Обновить"

```

#### Элементы

- Заголовок: "Написать @ник"

#### Действия
| Действие | Результат |
|----------|-----------|

| Редактирование текста | Свободное редактирование |
| MainButton | `openTelegramLink` / `openLink`, статус → "Написал", toast "Чат открыт", закрытие sheet |
| Drag вниз / тап вне | Закрыть sheet |

#### Переходы
```
MainButton      → открывается внешний чат + sheet закрывается
                  статус лида обновляется на "Написал"
Тап вне sheet   → sheet закрывается, остаёмся на Экране 2
```

## Все переходы между экранами

```
Старт (первый раз)
  └─→ Экран 0 (Онбординг)
        └─→ MainButton → Экран 1

Старт (повторный)
  └─→ Экран 1 (Лента) — сразу

Экран 1 (Лента)
  ├─→ тап карточки → Экран 2 (push)
  ├─→ тап "Написать" → Экран 3 (bottom sheet)
  ├─→ тап 🔍 → Экран 4 (push)
  ├─→ тап ⚙️ → Экран 5 (push)
  └─→ таб "Горячие" → Экран 6 (tab switch)

Экран 2 (Детали)
  ├─→ BackButton → Экран 1 (pop)
  └─→ MainButton → Экран 3 (bottom sheet)

Экран 3 (Sheet: Написать)
  ├─→ MainButton → внешний чат + закрыть sheet
  └─→ drag/тап вне → закрыть sheet

Экран 4 (Поиск)
  ├─→ BackButton → Экран 1 (pop)
  └─→ тап результата → Экран 2 (push)

Экран 5 (Настройки)
  ├─→ BackButton → Экран 1 (pop)
  └─→ MainButton "Сохранить" → Экран 1 (pop)

Экран 6 (Горячие)
  ├─→ тап карточки → Экран 2 (push)
  └─→ таб "Лента" → Экран 1 (tab switch)
```

## Что НЕ будет в версии 1.0

### Функциональные ограничения
| Что | Почему не сейчас |
|-----|-----------------|
| AI-фильтрация спама | Требует бэкенда с ML, сделаем v2 |
| Шаблоны с переменными | Сначала проверим нужны ли вообще |
| Аналитика / графики | Нужна статистика использования сначала |
| Реферальная программа | После стабильного retention |
| Swipe-to-action | Сложная жест-механика, упростим MVP |
| Автоответы | Требует Telegram Bot API сессии |
| Командная работа (агентство) | V3 |
| Платёжная система | Пока бесплатно, проверяем гипотезы |
| Тёмная тема (кастомная) | Telegram сам управляет темой |
| Push-уведомления от бота | V1.5 — после релиза Mini App |

### Упрощения в v1
- Вместо AI-настройки → ручной ввод ключевых слов
- Вместо дедупликации → просто показываем все (добавим в v1.1)
- Вместо реального парсинга → тестовые данные (моки)

## Технические требования к запуску

```javascript
// Обязательно при старте (index.tsx или App.tsx)
const tg = window.Telegram?.WebApp;
tg?.expand();       // развернуть на весь экран
tg?.ready();        // убрать нативный loader
tg?.setHeaderColor('secondary_bg_color'); // тема хедера
```

```css
/* Обязательно для всех экранов */
body {
  padding-bottom: env(safe-area-inset-bottom); /* iOS вырез */
  background-color: var(--tg-theme-bg-color);
  color: var(--tg-theme-text-color);
}
```

### Минимальные touch targets
- Кнопки: min-height 44px
- Строки списка: min-height 48px
- Иконки-кнопки: min 44×44px область нажатия (через padding)

## Порядок разработки (приоритет)

```
Sprint 1 — Ядро (можно показать пользователям)
  ├── Экран 0: Онбординг
  ├── Экран 1: Лента лидов (мок-данные)
  └── Экран 2: Детали лида

Sprint 2 — Главная ценность
  ├── Экран 3: Sheet "Написать"
  ├── Haptic feedback везде
  └── Фильтр-чипы в ленте

Sprint 3 — Удержание
  ├── Экран 5: Настройки
  ├── Экран 6: Горячие лиды
  └── Статусы лидов

Sprint 4 — Полировка
  ├── Экран 4: Поиск
  ├── Skeleton-загрузка
  ├── Empty states
  └── Pull-to-refresh
```
